# 09 - 案例：微服务架构

本案例演示如何在 K8s 上部署一套完整的微服务系统，涵盖服务拆分、服务间通信、配置管理、链路追踪和可观测性。

## 架构概览

```
                         ┌─────────────┐
          用户请求 ────► │   Ingress   │
                         └──────┬──────┘
                                │
              ┌─────────────────┼─────────────────┐
              ▼                 ▼                  ▼
    ┌──────────────┐  ┌──────────────┐   ┌──────────────┐
    │  API Gateway │  │   Frontend   │   │  Admin UI    │
    │  (Kong)      │  │   (React)    │   │  (Vue)       │
    └──────┬───────┘  └──────────────┘   └──────────────┘
           │
     ┌─────┼─────────┬──────────┐
     ▼     ▼         ▼          ▼
  ┌─────┐┌─────┐ ┌─────┐  ┌────────┐
  │User ││Order│ │Prod │  │Payment │
  │Svc  ││Svc  │ │Svc  │  │Svc     │
  └──┬──┘└──┬──┘ └──┬──┘  └───┬────┘
     │      │       │          │
     ▼      ▼       ▼          ▼
  ┌─────┐┌─────┐ ┌─────┐  ┌────────┐
  │User ││Order│ │Prod │  │Payment │
  │DB   ││DB   │ │DB   │  │DB      │
  └─────┘└─────┘ └─────┘  └────────┘
```

**技术选型：**
- 服务通信：gRPC（内部） + REST（对外）
- 配置管理：ConfigMap + 统一配置服务
- 服务发现：K8s DNS（无需额外注册中心）
- 可观测性：Prometheus + Grafana + Jaeger

## 第二步：统一配置管理

将公共配置集中管理，各微服务按需引用。

```yaml
# 02-common-config.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: common-config
  namespace: microservices
data:
  # 公共配置
  LOG_LEVEL: "info"
  LOG_FORMAT: "json"
  TRACING_ENABLED: "true"
  JAEGER_ENDPOINT: "http://jaeger-collector.observability.svc.cluster.local:14268/api/traces"

  # 服务注册（通过 K8s DNS，无需额外注册中心）
  USER_SVC: "user-svc.microservices.svc.cluster.local"
  ORDER_SVC: "order-svc.microservices.svc.cluster.local"
  PRODUCT_SVC: "product-svc.microservices.svc.cluster.local"
  PAYMENT_SVC: "payment-svc.microservices.svc.cluster.local"
---
apiVersion: v1
kind: Secret
metadata:
  name: db-credentials
  namespace: microservices
type: Opaque
stringData:
  USER_DB_URL: "mysql://app:AppP@ss@mysql-rw.webapp.svc.cluster.local:3306/userdb"
  ORDER_DB_URL: "mysql://app:AppP@ss@mysql-rw.webapp.svc.cluster.local:3306/orderdb"
  PRODUCT_DB_URL: "mysql://app:AppP@ss@mysql-rw.webapp.svc.cluster.local:3306/productdb"
  PAYMENT_DB_URL: "mysql://app:AppP@ss@mysql-rw.webapp.svc.cluster.local:3306/paymentdb"
```

## 第三步：微服务部署模板

以下展示 **用户服务** 的完整配置，其他服务结构类似。

### 用户服务（User Service）

```yaml
# 03-user-service.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: user-svc
  namespace: microservices
  labels:
    app: user-svc
    version: v1
spec:
  replicas: 2
  selector:
    matchLabels:
      app: user-svc
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  template:
    metadata:
      labels:
        app: user-svc
        version: v1
      annotations:
        prometheus.io/scrape: "true"
        prometheus.io/port: "8080"
        prometheus.io/path: "/metrics"
    spec:
      # ServiceAccount（用于服务间认证）
      serviceAccountName: user-svc-sa
      containers:
      - name: user-svc
        image: myregistry/user-svc:1.0.0
        ports:
        - containerPort: 8080
          name: http
        - containerPort: 9090
          name: grpc
        - containerPort: 8081
          name: metrics

        # 注入公共配置
        envFrom:
        - configMapRef:
            name: common-config
        # 注入数据库连接（敏感信息）
        env:
        - name: DB_URL
          valueFrom:
            secretKeyRef:
              name: db-credentials
              key: USER_DB_URL
        - name: REDIS_URL
          value: "redis://redis-svc.middleware.svc.cluster.local:6379/0"

        resources:
          requests:
            cpu: "100m"
            memory: "128Mi"
          limits:
            cpu: "500m"
            memory: "256Mi"

        readinessProbe:
          httpGet:
            path: /health
            port: 8080
          periodSeconds: 5
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 10
---
apiVersion: v1
kind: Service
metadata:
  name: user-svc
  namespace: microservices
spec:
  selector:
    app: user-svc
  ports:
  - name: http
    port: 80
    targetPort: 8080
  - name: grpc
    port: 9090
    targetPort: 9090
```

## 第四步：服务间通信

### 方式一：HTTP/gRPC 直连（通过 K8s DNS）

```python
# 订单服务中调用用户服务
import os
import requests

USER_SVC = os.environ.get("USER_SVC", "user-svc.microservices.svc.cluster.local")

def get_user(user_id):
    resp = requests.get(f"http://{USER_SVC}/api/users/{user_id}")
    return resp.json()
```

### 方式二：消息队列异步通信

```yaml
# 05-rabbitmq.yaml（中间件 namespace）
apiVersion: apps/v1
kind: Deployment
metadata:
  name: rabbitmq
  namespace: middleware
spec:
  replicas: 1
  selector:
    matchLabels:
      app: rabbitmq
  template:
    metadata:
      labels:
        app: rabbitmq
    spec:
      containers:
      - name: rabbitmq
        image: rabbitmq:3-management-alpine
        ports:
        - containerPort: 5672
          name: amqp
        - containerPort: 15672
          name: management
        env:
        - name: RABBITMQ_DEFAULT_USER
          valueFrom:
            secretKeyRef:
              name: rabbitmq-secret
              key: username
        - name: RABBITMQ_DEFAULT_PASS
          valueFrom:
            secretKeyRef:
              name: rabbitmq-secret
              key: password
        volumeMounts:
        - name: data
          mountPath: /var/lib/rabbitmq
      volumes:
      - name: data
        persistentVolumeClaim:
          claimName: rabbitmq-pvc
---
apiVersion: v1
kind: Service
metadata:
  name: rabbitmq-svc
  namespace: middleware
spec:
  selector:
    app: rabbitmq
  ports:
  - name: amqp
    port: 5672
  - name: management
    port: 15672
```

订单创建后发送事件，支付服务异步消费：

```python
# 订单服务：创建订单后发布事件
def create_order(order_data):
    order = save_to_db(order_data)
    # 异步通知支付服务
    channel.basic_publish(
        exchange='orders',
        routing_key='order.created',
        body=json.dumps({'order_id': order.id, 'amount': order.total})
    )
    return order

# 支付服务：消费订单事件
def on_order_created(message):
    data = json.loads(message)
    process_payment(data['order_id'], data['amount'])
```

## 第五步：Ingress 网关

```yaml
# 06-ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: microservices-ingress
  namespace: microservices
  annotations:
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
    nginx.ingress.kubernetes.io/proxy-body-size: "20m"
    # 限流
    nginx.ingress.kubernetes.io/limit-rps: "50"
    # 跨域
    nginx.ingress.kubernetes.io/enable-cors: "true"
    nginx.ingress.kubernetes.io/cors-allow-origin: "https://app.example.com"
    # 请求追踪 header 透传
    nginx.ingress.kubernetes.io/configuration-snippet: |
      proxy_set_header X-Request-ID $req_id;
      proxy_set_header X-Trace-ID $req_id;
spec:
  ingressClassName: nginx
  tls:
  - hosts:
    - api.example.com
    secretName: api-tls
  rules:
  - host: api.example.com
    http:
      paths:
      - path: /api/users
        pathType: Prefix
        backend:
          service:
            name: user-svc
            port:
              number: 80
      - path: /api/orders
        pathType: Prefix
        backend:
          service:
            name: order-svc
            port:
              number: 80
      - path: /api/products
        pathType: Prefix
        backend:
          service:
            name: product-svc
            port:
              number: 80
```

## 第六步：可观测性

### Prometheus 指标采集

```yaml
# 07-prometheus-config.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: prometheus-config
  namespace: observability
data:
  prometheus.yml: |
    global:
      scrape_interval: 15s

    scrape_configs:
    # 自动发现带 prometheus.io 注解的 Pod
    - job_name: 'kubernetes-pods'
      kubernetes_sd_configs:
      - role: pod
      relabel_configs:
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_scrape]
        action: keep
        regex: true
      - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_port]
        action: replace
        target_label: __address__
        regex: (.+)
        replacement: $1
      - source_labels: [__meta_kubernetes_namespace]
        action: replace
        target_label: namespace
      - source_labels: [__meta_kubernetes_pod_name]
        action: replace
        target_label: pod

    # 采集 kubelet 指标
    - job_name: 'kubelet'
      kubernetes_sd_configs:
      - role: node
      scheme: https
      tls_config:
        ca_file: /var/run/secrets/kubernetes.io/serviceaccount/ca.crt
```

### 服务暴露指标

每个微服务通过 `/metrics` 端点暴露 Prometheus 格式指标：

```python
# Python Flask 示例
from prometheus_flask_exporter import PrometheusMetrics

app = Flask(__name__)
metrics = PrometheusMetrics(app)

# 自动暴露：请求总数、延迟分布、状态码分布
# http_requests_total{method="GET",endpoint="/api/users",status="200"}
# http_request_duration_seconds_bucket{le="0.1",...}
```

## 第七步：RBAC 与服务间认证

```yaml
# 10-rbac.yaml
# 每个微服务独立的 ServiceAccount
apiVersion: v1
kind: ServiceAccount
metadata:
  name: user-svc-sa
  namespace: microservices
---
apiVersion: v1
kind: ServiceAccount
metadata:
  name: order-svc-sa
  namespace: microservices
---
# 限制 order-svc 只能访问特定资源
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: order-svc-role
  namespace: microservices
rules:
- apiGroups: [""]
  resources: ["configmaps"]
  resourceNames: ["common-config"]
  verbs: ["get"]
---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: order-svc-binding
  namespace: microservices
subjects:
- kind: ServiceAccount
  name: order-svc-sa
roleRef:
  kind: Role
  name: order-svc-role
  apiGroup: rbac.authorization.k8s.io
```

## 微服务通信模式总结

| 模式 | 实现方式 | 适用场景 |
|------|---------|---------|
| **同步 HTTP** | K8s DNS + Service | 简单请求-响应，实时查询 |
| **同步 gRPC** | K8s DNS + Service (port 9090) | 高性能内部通信，强类型接口 |
| **异步消息** | RabbitMQ / Kafka | 解耦、削峰、最终一致性 |
| **事件驱动** | Kafka + Event Sourcing | 复杂事件流、审计日志 |

## 知识点回顾

| 使用的资源/概念 | 作用 |
|----------------|------|
| Namespace | 按职能隔离（服务/中间件/可观测性） |
| ConfigMap (common) | 统一公共配置，各服务共享引用 |
| Secret | 数据库连接串等敏感信息 |
| Service (ClusterIP) | 服务发现与负载均衡 |
| Ingress | 外部流量入口与路由 |
| ServiceAccount + RBAC | 服务间认证与权限控制 |
| Prometheus + Grafana | 指标采集与可视化 |
| Jaeger | 分布式链路追踪 |
