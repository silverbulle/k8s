# 03 - 工作负载资源

工作负载（Workload）资源定义了应用如何在集群中运行。K8s 提供了一套层次化的资源抽象，从最小的 Pod 到高级的 Deployment、StatefulSet 等控制器。

## 资源层次关系

```
Controller（期望状态）
  │
  ├── Deployment ──► ReplicaSet ──► Pod（无状态应用）
  ├── StatefulSet ──────────────► Pod（有状态应用）
  ├── DaemonSet ────────────────► Pod（每节点一个）
  ├── Job ──────────────────────► Pod（一次性任务）
  └── CronJob ──► Job ──────────► Pod（定时任务）
```

**核心理念：控制器不直接管理 Pod，而是通过声明式 API 维护期望状态。** 实际状态由 kubelet 在节点上执行。

## 3.1 Pod

Pod 是 K8s 最小的可部署单元，是一组共享网络和存储的容器集合。

### Pod 的本质

```
┌─────────────────────────────────────┐
│                 Pod                  │
│                                     │
│  IP: 10.244.1.5 (共享网络命名空间)   │
│                                     │
│  ┌─────────┐  ┌─────────┐          │
│  │  App    │  │ Sidecar │          │
│  │ :8080   │◄─┤ :9090   │          │
│  └─────────┘  └─────────┘          │
│                                     │
│  ┌──────────────────────────┐      │
│  │  Volume (共享存储)        │      │
│  └──────────────────────────┘      │
└─────────────────────────────────────┘
```

- 同一 Pod 中的容器**共享网络命名空间**（相同 IP、可通过 `localhost` 通信）
- 通过 Volume **共享存储**
- Pod 是**临时性的**（ephemeral），随时可能被创建和销毁

### Pod 配置示例

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp
  labels:
    app: myapp
    version: v1
spec:
  # 初始化容器（在主容器启动前运行完毕）
  initContainers:
  - name: init-db
    image: busybox
    command: ['sh', '-c', 'until nslookup mysql; do sleep 2; done']

  containers:
  - name: app
    image: myapp:1.0
    ports:
    - containerPort: 8080

    # 资源限制
    resources:
      requests:                  # 调度依据（最低保证）
        cpu: "100m"
        memory: "128Mi"
      limits:                    # 上限（超出则 throttle/OOM）
        cpu: "500m"
        memory: "256Mi"

    # 环境变量
    env:
    - name: DB_HOST
      value: "mysql.default.svc.cluster.local"
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: db-secret
          key: password

    # Volume 挂载
    volumeMounts:
    - name: config
      mountPath: /etc/config
      readOnly: true

  # Pod 级别配置
  restartPolicy: Always          # Always | OnFailure | Never
  terminationGracePeriodSeconds: 30

  volumes:
  - name: config
    configMap:
      name: app-config
```

### QoS 等级

Pod 的资源配置决定了其 QoS（Quality of Service）等级，影响节点压力时的驱逐优先级：

| QoS 等级 | 条件 | 驱逐优先级 |
|----------|------|-----------|
| **Guaranteed** | requests == limits（所有容器都设置） | 最低（最后驱逐） |
| **Burstable** | 至少一个容器设了 requests 或 limits | 中 |
| **BestEffort** | 没有设置任何 requests/limits | 最高（最先驱逐） |

### 常见 Pod 模式

**Sidecar 模式** — 辅助容器增强主容器能力：

```yaml
containers:
- name: app            # 主业务容器
  image: myapp:1.0
- name: log-collector  # 日志收集 sidecar
  image: fluentd:latest
  volumeMounts:
  - name: app-logs
    mountPath: /var/log/app
```

**Ambassador 模式** — 代理外部访问：

```yaml
containers:
- name: app           # 主业务容器，访问 localhost:6379
  image: myapp:1.0
- name: redis-proxy   # 代理到远程 Redis
  image: envoy:latest
```

## 3.2 ReplicaSet

ReplicaSet 确保任意时刻运行指定数量的 Pod 副本。

```yaml
apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: myapp-rs
spec:
  replicas: 3
  selector:
    matchLabels:
      app: myapp
  template:                    # Pod 模板
    metadata:
      labels:
        app: myapp
    spec:
      containers:
      - name: app
        image: myapp:1.0
```

**实际使用中很少直接创建 ReplicaSet**，通常通过 Deployment 间接管理。Deployment 提供了滚动更新、回滚等 ReplicaSet 不具备的能力。

### 工作原理

```
ReplicaSet 控制循环:

期望: 3 个 Pod (replicas=3)
实际: 2 个 Pod 运行中
  → 创建 1 个新 Pod

实际: 4 个 Pod 运行中
  → 删除 1 个多余的 Pod（优先删除最老的）
```

## 3.3 Deployment

Deployment 是**无状态应用**最常用的工作负载控制器，管理 ReplicaSet 来实现声明式更新。

### 核心能力

| 能力 | 说明 |
|------|------|
| **声明式更新** | 修改 Pod 模板，自动触发滚动更新 |
| **回滚** | 一键回滚到任意历史版本 |
| **扩缩容** | 修改 replicas 字段 |
| **暂停/恢复** | 批量修改后一次性发布 |

### 配置示例

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: webapp
spec:
  replicas: 5
  selector:
    matchLabels:
      app: webapp
  
  # 滚动更新策略
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1               # 更新时最多多出 1 个 Pod
      maxUnavailable: 0          # 更新时不允许有 Pod 不可用

  template:
    metadata:
      labels:
        app: webapp
    spec:
      containers:
      - name: webapp
        image: webapp:2.0
        ports:
        - containerPort: 8080
        readinessProbe:
          httpGet:
            path: /ready
            port: 8080
          periodSeconds: 5
```

### 滚动更新过程

```
初始状态:
  RS-old:  [Pod-1] [Pod-2] [Pod-3]     ← 旧版本
  RS-new:  (空)                         ← 新版本

更新中 (maxSurge=1, maxUnavailable=0):
  RS-old:  [Pod-1] [Pod-2]              ← 旧 RS 缩容
  RS-new:  [Pod-4] [Pod-5] [Pod-6]      ← 新 RS 扩容

更新完成:
  RS-old:  (保留，replicas=0)            ← 历史版本，用于回滚
  RS-new:  [Pod-4] [Pod-5] [Pod-6]      ← 当前版本
```

### 常用操作

```bash
# 创建/更新
kubectl apply -f deployment.yaml

# 查看滚动更新状态
kubectl rollout status deployment/webapp

# 查看更新历史
kubectl rollout history deployment/webapp

# 回滚到上一版本
kubectl rollout undo deployment/webapp

# 回滚到指定版本
kubectl rollout undo deployment/webapp --to-revision=3

# 暂停/恢复（批量修改）
kubectl rollout pause deployment/webapp
kubectl set image deployment/webapp webapp=webapp:3.0
kubectl set resources deployment/webapp --limits=cpu=500m,memory=256Mi
kubectl rollout resume deployment/webapp    # 恢复后一次性触发更新

# 扩缩容
kubectl scale deployment/webapp --replicas=10
```

## 3.4 StatefulSet

StatefulSet 专为**有状态应用**设计，提供稳定的网络标识、稳定的存储和有序部署。

### 与 Deployment 的关键区别

| 特性 | Deployment | StatefulSet |
|------|-----------|-------------|
| Pod 名称 | 随机后缀 | 有序编号（app-0, app-1, app-2） |
| 创建顺序 | 并行 | 有序（0→1→2，前一个 Ready 后才创建下一个） |
| 网络标识 | 不稳定 | 稳定（Headless Service + Pod DNS） |
| 存储 | 共享 PVC | 每个 Pod 独立 PVC |
| 扩缩容 | 任意 | 有序（缩容从最大编号开始） |
| 删除行为 | 删除所有 Pod | 保留 PVC（需手动清理） |

### 配置示例

```yaml
apiVersion: apps/v1
kind: StatefulSet
metadata:
  name: mysql
spec:
  serviceName: mysql-headless    # 必须配合 Headless Service
  replicas: 3
  selector:
    matchLabels:
      app: mysql
  template:
    metadata:
      labels:
        app: mysql
    spec:
      containers:
      - name: mysql
        image: mysql:8.0
        ports:
        - containerPort: 3306
        env:
        - name: MYSQL_ROOT_PASSWORD
          valueFrom:
            secretKeyRef:
              name: mysql-secret
              key: root-password
        volumeMounts:
        - name: data
          mountPath: /var/lib/mysql

  # 每个 Pod 独立的 PVC 模板
  volumeClaimTemplates:
  - metadata:
      name: data
    spec:
      accessModes: ["ReadWriteOnce"]
      storageClassName: standard
      resources:
        requests:
          storage: 10Gi
```

### 稳定网络标识

```
# StatefulSet + Headless Service 提供稳定 DNS
mysql-0.mysql-headless.default.svc.cluster.local
mysql-1.mysql-headless.default.svc.cluster.local
mysql-2.mysql-headless.default.svc.cluster.local

# Pod 重启后 DNS 不变（Deployment 的 Pod DNS 会变）
```

### 有序部署与删除

```
扩容: mysql-0 (Ready) → mysql-1 (Ready) → mysql-2 (Ready)
缩容: mysql-2 (Terminating) → mysql-1 (Terminating) → mysql-0
```

## 3.5 DaemonSet

DaemonSet 确保**每个 Node（或满足条件的 Node）上运行恰好一个 Pod 副本**。

### 典型用途

- 日志收集（Fluentd, Filebeat）
- 节点监控（Node Exporter, Datadog Agent）
- 网络插件（Calico, Flannel, Cilium）
- 存储守护进程（Ceph, GlusterFS）

### 配置示例

```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: node-exporter
  namespace: monitoring
spec:
  selector:
    matchLabels:
      app: node-exporter
  template:
    metadata:
      labels:
        app: node-exporter
    spec:
      # 使用 hostNetwork 暴露节点端口
      hostNetwork: true
      # 容忍控制平面的污点，使其也能运行在 Master 节点
      tolerations:
      - key: node-role.kubernetes.io/control-plane
        effect: NoSchedule
      containers:
      - name: node-exporter
        image: prom/node-exporter:latest
        ports:
        - containerPort: 9100
          hostPort: 9100
        volumeMounts:
        - name: proc
          mountPath: /host/proc
          readOnly: true
      volumes:
      - name: proc
        hostPath:
          path: /proc
```

### 更新策略

```yaml
spec:
  updateStrategy:
    type: RollingUpdate
    rollingUpdate:
      maxUnavailable: 1           # 每次最多更新 1 个节点
      maxSurge: 0                 # 不允许额外 Pod
```

## 3.6 Job 与 CronJob

### Job — 一次性任务

确保指定数量的 Pod 成功完成（退出码为 0）。

```yaml
apiVersion: batch/v1
kind: Job
metadata:
  name: db-migration
spec:
  completions: 1                  # 总共需要成功完成的 Pod 数
  parallelism: 1                  # 最多并行运行的 Pod 数
  backoffLimit: 3                 # 最多重试 3 次
  activeDeadlineSeconds: 600      # 超时 10 分钟
  template:
    spec:
      restartPolicy: OnFailure    # Job 的 Pod 只能用 OnFailure 或 Never
      containers:
      - name: migrate
        image: myapp:1.0
        command: ["python", "manage.py", "migrate"]
```

**并行模式：**

```yaml
# 固定完成数（Work Queue 模式）
completions: 5
parallelism: 2
# 同时运行 2 个，总共完成 5 个

# 并行展开（每个 Pod 不同索引）
completions: 5
parallelism: 5
completionMode: Indexed
# Pod 获得索引 0-4，可根据索引处理不同数据
```

### CronJob — 定时任务

按 Cron 表达式定期创建 Job。

```yaml
apiVersion: batch/v1
kind: CronJob
metadata:
  name: db-backup
spec:
  schedule: "0 2 * * *"           # 每天凌晨 2 点
  concurrencyPolicy: Forbid       # 禁止并发执行
  successfulJobsHistoryLimit: 3   # 保留最近 3 个成功的 Job
  failedJobsHistoryLimit: 1       # 保留最近 1 个失败的 Job
  startingDeadlineSeconds: 200    # 错过后 200 秒内还可补执行
  jobTemplate:
    spec:
      template:
        spec:
          restartPolicy: OnFailure
          containers:
          - name: backup
            image: mysql:8.0
            command:
            - sh
            - -c
            - |
              mysqldump -h mysql -u root -p${MYSQL_ROOT_PASSWORD} --all-databases \
                | gzip > /backup/all-$(date +%Y%m%d).sql.gz
            volumeMounts:
            - name: backup-storage
              mountPath: /backup
          volumes:
          - name: backup-storage
            persistentVolumeClaim:
              claimName: backup-pvc
```

**CronJob 并发策略：**

| 策略 | 行为 |
|------|------|
| `Allow`（默认） | 允许并发，多个 Job 可同时运行 |
| `Forbid` | 跳过新的执行如果上一次还在运行 |
| `Replace` | 取消正在运行的 Job，启动新的 |

## 如何选择工作负载

| 场景 | 推荐资源 | 理由 |
|------|---------|------|
| Web 服务 / API 服务 | **Deployment** | 无状态，需要滚动更新和回滚 |
| 数据库 / 消息队列 | **StatefulSet** | 需要稳定标识和持久化存储 |
| 日志收集 / 监控 Agent | **DaemonSet** | 每个节点都需要一个实例 |
| 数据迁移 / ETL | **Job** | 运行完成即结束 |
| 定时备份 / 报表 | **CronJob** | 按周期执行 |
| 简单测试 / 调试 | **Pod**（直接创建） | 临时用途，不推荐生产使用 |

## 本章小结

| 资源 | 一句话总结 | 关键概念 |
|------|-----------|---------|
| Pod | 最小部署单元，容器集合 | 共享网络/存储, 探针, QoS |
| ReplicaSet | 维持副本数 | 标签选择器, 通常不直接使用 |
| Deployment | 无状态应用的标准控制器 | 滚动更新, 回滚, maxSurge/maxUnavailable |
| StatefulSet | 有状态应用控制器 | 有序编号, 稳定 DNS, 独立 PVC |
| DaemonSet | 每节点一个 Pod | 日志/监控/网络插件 |
| Job/CronJob | 一次性/定时任务 | completions, 并发策略, backoffLimit |
