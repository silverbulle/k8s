# 04 - 服务与网络

K8s 的网络模型解决了两个核心问题：**Pod 之间如何通信**（扁平网络）和**外部如何访问 Pod**（服务暴露）。本章聚焦服务发现和网络策略。

## K8s 网络基础

K8s 的网络设计遵循以下原则：

1. **每个 Pod 拥有独立 IP**（不需要 NAT）
2. **所有 Pod 之间可以直接通信**（扁平网络，无防火墙限制）
3. **Node 可以与该 Node 上的所有 Pod 通信**

```
┌──────────────────────────────────────────────┐
│              K8s Cluster                      │
│                                              │
│  Node-1                   Node-2             │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐        │
│  │ Pod A   │ │ Pod B   │ │ Pod C   │        │
│  │10.244   │ │10.244   │ │10.244   │        │
│  │ .1.5    │◄┤ .1.8    │◄┤ .2.3    │        │
│  └─────────┘ └─────────┘ └─────────┘        │
│                                              │
│  Service: 10.96.0.10 (虚拟 IP，通过 iptables 转发)
└──────────────────────────────────────────────┘
```

**Pod IP 是临时的**——Pod 重建后 IP 会变。Service 提供了稳定的访问入口。

## 4.1 Service

Service 是一组 Pod 的逻辑抽象，提供稳定的网络端点（IP + 端口），后端 Pod 变化时自动更新。

### Service 类型

| 类型 | 访问范围 | 实现方式 |
|------|---------|---------|
| **ClusterIP**（默认） | 集群内部 | 分配虚拟 IP，通过 iptables/IPVS 转发 |
| **NodePort** | 集群外部 | 在每个 Node 上开放固定端口（30000-32767） |
| **LoadBalancer** | 集群外部 | 云厂商负载均衡器 |
| **ExternalName** | 外部服务 | CNAME 映射到外部域名 |

### ClusterIP

```yaml
apiVersion: v1
kind: Service
metadata:
  name: webapp-svc
spec:
  type: ClusterIP
  selector:
    app: webapp              # 匹配带有 app=webapp 标签的 Pod
  ports:
  - port: 80                 # Service 端口
    targetPort: 8080          # Pod 实际端口
    protocol: TCP
```

```
集群内访问方式：
  curl http://webapp-svc.default.svc.cluster.local:80
  curl http://webapp-svc:80              # 同 namespace 可缩写
```

**Headless Service**（ClusterIP: None）：

```yaml
spec:
  clusterIP: None            # 不分配虚拟 IP
  selector:
    app: mysql
```

DNS 直接返回所有后端 Pod 的 IP 列表（而不是虚拟 IP），常用于 StatefulSet：

```
# DNS 查询返回所有 Pod IP
$ nslookup mysql-headless
Name: 10.244.1.5
Name: 10.244.2.8
Name: 10.244.3.2

# 也可以查询单个 Pod
$ nslookup mysql-0.mysql-headless
Name: 10.244.1.5
```

### NodePort

```yaml
apiVersion: v1
kind: Service
metadata:
  name: webapp-nodeport
spec:
  type: NodePort
  selector:
    app: webapp
  ports:
  - port: 80                  # Service 端口（集群内部访问）
    targetPort: 8080           # Pod 实际端口
    nodePort: 30080            # 每个 Node 上暴露的端口
```

```
外部访问方式：
  curl http://<任意Node-IP>:30080
```

### LoadBalancer

```yaml
apiVersion: v1
kind: Service
metadata:
  name: webapp-lb
spec:
  type: LoadBalancer
  selector:
    app: webapp
  ports:
  - port: 80
    targetPort: 8080
```

在云环境中，K8s 会自动创建一个云负载均衡器（AWS ELB/ALB、GCP LB、Azure LB）并将流量引入。

```bash
# 查看分配的外部 IP
kubectl get svc webapp-lb
NAME          TYPE           EXTERNAL-IP     PORT(S)
webapp-lb     LoadBalancer   52.1.2.3        80:31234/TCP
```

### ExternalName

```yaml
apiVersion: v1
kind: Service
metadata:
  name: external-db
spec:
  type: ExternalName
  externalName: db.example.com    # CNAME 映射
```

```
# 集群内访问 external-db → DNS 解析为 db.example.com
curl http://external-db.default.svc.cluster.local
# 实际访问 → http://db.example.com
```

### Service 与 Endpoints

```
Service (selector: app=webapp)
  │
  ├──► Endpoints 对象（自动维护）
  │      ├── 10.244.1.5:8080
  │      ├── 10.244.2.8:8080
  │      └── 10.244.3.2:8080
  │
  └── 只有 Readiness Probe 通过的 Pod 才会出现在 Endpoints 中
```

```bash
# 查看 Endpoints
kubectl get endpoints webapp-svc
NAME          ENDPOINTS                                  AGE
webapp-svc    10.244.1.5:8080,10.244.2.8:8080,...        5m
```

## 4.2 Ingress

Ingress 提供 HTTP/HTTPS 路由规则，将外部流量路由到不同的 Service。它工作在 L7（应用层），比 Service（L4）提供更丰富的路由能力。

### Ingress Controller

Ingress 资源需要配合 **Ingress Controller** 才能工作。常见的 Controller：

| Controller | 说明 |
|-----------|------|
| **NGINX Ingress** | 最常用，功能完善 |
| **Traefik** | 自动服务发现，配置简单 |
| **Kong** | 基于 OpenResty，插件丰富 |
| **AWS ALB Controller** | AWS 原生 ALB 集成 |

### 基础路由

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: webapp-ingress
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
spec:
  ingressClassName: nginx          # 指定使用哪个 Ingress Controller
  rules:
  - host: app.example.com
    http:
      paths:
      - path: /api
        pathType: Prefix
        backend:
          service:
            name: api-svc
            port:
              number: 80
      - path: /
        pathType: Prefix
        backend:
          service:
            name: frontend-svc
            port:
              number: 80
```

```
请求路由：
  app.example.com/api/*    → api-svc:80
  app.example.com/*        → frontend-svc:80
```

### TLS/HTTPS

```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: secure-ingress
spec:
  ingressClassName: nginx
  tls:
  - hosts:
    - app.example.com
    secretName: tls-secret           # 引用 TLS 证书 Secret
  rules:
  - host: app.example.com
    http:
      paths:
      - path: /
        pathType: Prefix
        backend:
          service:
            name: webapp-svc
            port:
              number: 443
```

创建 TLS Secret：

```bash
kubectl create secret tls tls-secret \
  --cert=tls.crt --key=tls.key
```

### 高级配置

```yaml
metadata:
  annotations:
    # 限流
    nginx.ingress.kubernetes.io/limit-rps: "10"
    # CORS
    nginx.ingress.kubernetes.io/enable-cors: "true"
    # 认证
    nginx.ingress.kubernetes.io/auth-type: basic
    nginx.ingress.kubernetes.io/auth-secret: basic-auth
    # WebSocket 支持
    nginx.ingress.kubernetes.io/proxy-read-timeout: "3600"
    nginx.ingress.kubernetes.io/proxy-send-timeout: "3600"
    # SSL 重定向
    nginx.ingress.kubernetes.io/ssl-redirect: "true"
    # 请求体大小
    nginx.ingress.kubernetes.io/proxy-body-size: "50m"
```

## 4.3 DNS（CoreDNS）

K8s 内置 DNS 服务（CoreDNS），为 Service 和 Pod 提供域名解析。

### DNS 记录格式

```
# Service DNS
<service-name>.<namespace>.svc.cluster.local

# Pod DNS（需要 StatefulSet + Headless Service）
<pod-name>.<headless-svc-name>.<namespace>.svc.cluster.local

# 示例
webapp-svc.default.svc.cluster.local     → Service ClusterIP
mysql-0.mysql-headless.prod.svc.cluster.local → 特定 Pod IP
```

### DNS 查询流程

```
Pod 发起 DNS 查询
  │
  ├─ /etc/resolv.conf 配置
  │   nameserver 10.96.0.10      ← CoreDNS Service IP
  │   search default.svc.cluster.local svc.cluster.local cluster.local
  │
  └─→ CoreDNS 解析
       ├─ K8s Service → 返回 ClusterIP
       ├─ K8s Headless Service → 返回 Pod IP 列表
       └─ 外部域名 → 转发到上游 DNS（如 8.8.8.8）
```

### CoreDNS 配置（Corefile）

```bash
# 查看 CoreDNS 配置
kubectl get configmap coredns -n kube-system -o yaml
```

```yaml
# Corefile 核心配置
.:53 {
    errors                          # 错误日志
    health                          # 健康检查端点
    ready                           # 就绪端点
    kubernetes cluster.local in-addr.arpa ip6.arpa {
       pods insecure                # Pod DNS 记录策略
       fallthrough in-addr.arpa ip6.arpa
    }
    prometheus :9153                # 指标暴露
    forward . /etc/resolv.conf      # 外部域名转发
    cache 30                        # 缓存 30 秒
    loop                            # 检测转发循环
    reload                          # 自动重载配置
    loadbalance                     # 轮询负载均衡
}
```

## 4.4 NetworkPolicy

NetworkPolicy 控制 Pod 之间的网络流量，实现网络隔离。需要 CNI 插件支持（Calico、Cilium 等，Flannel 默认不支持）。

### 默认行为

- 没有 NetworkPolicy 时：所有 Pod 可以互相通信（无限制）
- 一旦为某个 Pod 创建了 NetworkPolicy：只允许策略中定义的流量

### 示例：限制 Web 只能被 API 访问

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: web-access
  namespace: default
spec:
  podSelector:
    matchLabels:
      app: web              # 作用于 app=web 的 Pod
  policyTypes:
  - Ingress                 # 入站规则
  ingress:
  - from:
    - podSelector:
        matchLabels:
          app: api           # 只允许 app=api 的 Pod 访问
    ports:
    - protocol: TCP
      port: 8080
```

### 示例：隔离 namespace

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: deny-other-namespaces
  namespace: production
spec:
  podSelector: {}            # 作用于该 namespace 下所有 Pod
  policyTypes:
  - Ingress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          name: production   # 只允许同 namespace
    - podSelector: {}        # 同 namespace 的所有 Pod
```

### 示例：允许出站但限制目标

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: restrict-egress
spec:
  podSelector:
    matchLabels:
      app: backend
  policyTypes:
  - Egress
  egress:
  - to:
    - podSelector:
        matchLabels:
          app: database
    ports:
    - protocol: TCP
      port: 3306             # 只能访问 MySQL
  - to:
    - namespaceSelector: {}
    ports:
    - protocol: UDP
      port: 53               # 允许 DNS 查询
```
